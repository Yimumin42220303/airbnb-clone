# AI 숙소 추천 고도화 제안

## 사용자 아이디어 (반영)

1. **전용 URL**  
   하단 네비 "AI숙소추천" 탭 시 **플랫폼 내 전용 페이지**(`/recommend`)로 이동해, 한 화면에서 세부 니즈를 입력할 수 있게 함.

2. **여행 유형**  
   - 친구 / 커플 / 가족 중 선택  
   - AI가 동행 구성에 맞는 숙소(넓이, 침실 수, 분위기 등)를 고려해 추천.

3. **우선순위**  
   - 가성비 중시 / 평점 중시 / 위치 중시 중 선택  
   - 추천 시 가격·리뷰·역/지역 접근성을 가중치에 반영.

4. **자유 입력**  
   - 그 외 선호사항을 텍스트로 입력 (예: "시부야 근처", "주방 필수", "조용한 동네")  
   - 기존 `preferences` 필드와 동일하게 AI 프롬프트에 포함.

## 추가 제안 (게스트 경험 향상)

### 1. 숙소 유형 (선택)
- **아파트 1실** / **단독주택·독채** / **게스트하우스** 등  
- `propertyType`과 매칭해 “가족은 단독주택”, “1인은 게스트하우스” 등 추천 정확도 향상.

### 2. 여행 목적 (선택)
- **관광** / **출장** / **장기 체류(1주 이상)**  
- 관광→역 근처·관광지 접근, 출장→와이파이·작업 공간, 장기→주방·세탁 강조 등으로 프롬프트 보강.

### 3. 필수 편의시설 (다중 선택)
- **주방** / **세탁기** / **와이파이** / **에어컨** / **주차** 등 체크박스  
- 후보 리스팅을 `amenities`로 이미 가져오고 있으므로, AI에게 “이 중 필수는 O, O”라고만 전달해도 됨.

### 4. 예산 힌트 (선택)
- **~10만원대** / **20만원대** / **30만원대 이상** / **상관없음**  
- 1박 기준 힌트로, AI가 “가성비 중시”와 조합해 적정 구간 강조.

### 5. 이동 수단 (선택)
- **전철 위주** / **택시·도보 병행**  
- “역 도보 5분 이내” vs “조금 떨어져도 괜찮음” 등 위치·접근성 설명을 프롬프트에 추가.

### 6. UX 보완
- **저장/공유**: “이 조건으로 다시 추천” 링크(같은 쿼리 유지) 또는 결과 URL 공유.  
- **결과 정렬**: “가격순 / 평점순 / 추천순” 토글 (API는 그대로 두고 프론트 정렬만).  
- **빈 결과 시**: “조건 완화” 안내 문구 + 검색 페이지로 가는 CTA.

## 구현 범위 (1차)

- 전용 **`/recommend`** 페이지 (리다이렉트 제거, 폼 전면 노출).  
- 입력 항목: **여행 유형**(친구/커플/가족), **우선순위**(가성비/평점/위치), **날짜·인원**, **선호사항(자유 입력)**.  
- **API 확장**: `tripType`, `priority` 추가 후 시스템/유저 프롬프트에 반영.

추가 제안(숙소 유형, 여행 목적, 필수 편의시설, 예산, 이동 수단)은 2차에서 단계적으로 넣을 수 있음.
