# AI 추천 · 우선순위 최대 3개 선택 UX 제안

## 요구사항
「무엇을 가장 중요하게 보시나요?」 섹션에서 **최대 3개**까지 선택 가능하도록 제한.

---

## 제안한 UX/UI (구현 반영)

### 1. 선택 개수 표시
- 제목 아래에 **「최대 3개 선택 · 0/3 선택」** 문구 추가.
- 선택할 때마다 **1/3**, **2/3**, **3/3**으로 갱신되어, 남은 선택 가능 개수를 바로 알 수 있음.

### 2. 3개 선택 시 나머지 비활성화
- **3개를 선택한 뒤**에는, 선택되지 않은 버튼을 **비활성화(회색, 클릭 불가)** 처리.
- 더 이상 선택할 수 없다는 것이 시각적으로 명확함.
- 4번째를 눌러도 반응하지 않으므로, 토스트 없이도 이해 가능.

### 3. 선택 해제
- **이미 선택된 항목을 다시 클릭**하면 해제됨.
- 해제 후에는 다른 항목을 선택할 수 있어, 3개 이내로 자유롭게 바꿀 수 있음.

### 4. 시각적 상태
- **선택됨**: 기존과 동일하게 빨간 배경 + 흰 글씨.
- **선택 가능(미선택)**: 흰 배경 + 테두리, 호버 시 강조.
- **비활성**: 연한 회색 배경 + 회색 글씨, `cursor-not-allowed`.

### 5. API
- 요청 본문에 **priorities: string[]** (최대 3개) 전달.
- 기존 단일 **priority**도 그대로 지원(하위 호환).  
- AI 프롬프트에는 「가성비, 평점, 위치」처럼 쉼표로 이어서 전달.

---

## 정리
- **카운터(0/3~3/3)**로 제한을 인지시키고,
- **3개 초과 시 나머지 비활성화**로 추가 선택을 막으며,
- **선택 해제**로 조합을 쉽게 바꿀 수 있게 한 구성.
